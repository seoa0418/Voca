영어 단어 학습을 위한 안드로이드 앱

1. 앱 소개

Voca는 영어 단어를 무작위로 하나씩 제시하고,
사용자가 스스로 뜻을 떠올린 뒤 확인할 수 있도록 설계한
간단하지만 학습 효과를 고려한 영어 단어 학습 앱입니다.

단순히 단어와 뜻을 동시에 보여주는 방식이 아니라,
뜻을 숨긴 상태에서 먼저 생각해 보고,
필요할 때 직접 뜻을 확인하도록 만들어
능동적인 단어 암기를 목표로 했습니다.

2. 앱 개발 배경 (의도)

영어 단어를 외울 때 가장 흔한 방식은
단어와 뜻을 동시에 보고 반복해서 읽는 방법입니다.
하지만 이 방식은 단어를 “아는 것처럼 착각”하게 만들고,
실제로는 잘 기억나지 않는 경우가 많았습니다.

그래서 저는

“단어를 보고 → 스스로 뜻을 생각한 뒤 → 확인하는 구조”

를 가진 앱을 직접 만들어 보고 싶었습니다.

또한 안드로이드 수업에서 배운

Jetpack Compose

ViewModel

API 연동

비동기 처리

를 실제로 모두 활용해 볼 수 있는
작지만 완성도 있는 앱을 만드는 것을 목표로 삼았습니다.

3. 이 앱이 특별한 이유 (나만의 요소)

이 앱은 단순한 예제 앱이 아니라,
다음과 같은 점에서 저만의 의도가 담긴 앱입니다.

단어를 바로 보여주지 않고 뜻을 숨기는 학습 구조

버튼 클릭으로 사용자가 학습 흐름을 직접 조절

랜덤 단어 선택으로 매번 다른 학습 경험 제공

실제 외부 API를 연동하여 실사용 가능한 앱 구조

특히 “뜻 보기 / 뜻 숨기기” 기능은
단순 UI 기능이 아니라 학습 방식 자체를 고려한 설계입니다.

4. 앱의 전체 구조

이 앱은 MVVM 아키텍처를 기반으로 구성되어 있습니다.

UI (Jetpack Compose)
 └── MainActivity / VocabularyScreen
        ↓ (사용자 입력)
ViewModel (WordViewModel)
        ↓ (상태 관리)
WordUiState
        ↓ (데이터 요청)
외부 API (Dictionary API, Translation API)

구조를 이렇게 만든 이유

UI와 로직을 분리해 코드 가독성 향상

화면 회전 등에도 데이터 유지

상태 변화에 따라 UI 자동 갱신

6. 앱의 주요 기능
1️⃣ 랜덤 영어 단어 제공

자주 사용되는 영어 단어 목록 중 하나를 무작위로 선택

앱 실행 시 자동으로 첫 단어 로딩

2️⃣ 뜻 숨기기 / 보기 기능

처음에는 뜻이 보이지 않음

사용자가 버튼을 눌러 직접 확인

단어가 바뀌면 다시 뜻 숨김 상태로 초기화

3️⃣ 다음 단어 불러오기

버튼 클릭 시 새로운 단어 로딩

이전 단어와 겹치지 않는 랜덤 학습 경험 제공

4️⃣ 외부 API 연동

Dictionary API: 영어 단어 정보 확인

Translation API: 영어 → 한국어 번역

7. API 연동 설명 (어떻게 만들었는가)

이 앱은 실제 외부 REST API와 통신합니다.

📌 Dictionary API

영어 단어가 실제 존재하는지 확인

단어의 기본적인 의미 정보 제공

요청 예시:

https://api.dictionaryapi.dev/api/v2/entries/en/word

📌 Translation API (MyMemory)

영어 단어를 한국어로 번역

GET 방식으로 쿼리 파라미터 전달

요청 예시:

https://api.mymemory.translated.net/get?q=hello&langpair=en|ko

8. 비동기 처리와 상태 관리

네트워크 통신은 시간이 걸리기 때문에
메인 스레드에서 실행하면 앱이 멈추게 됩니다.

이를 해결하기 위해

Coroutine

viewModelScope

를 사용하여 비동기 처리를 구현했습니다.

viewModelScope.launch {
    // API 호출
    // 결과를 UI 상태로 반영
}


API 호출 결과는 WordUiState에 저장되고,
상태가 바뀌면 UI는 자동으로 다시 그려집니다.

9. UI 설계 의도

카드(Card) UI로 단어를 강조

버튼을 화면 하단에 배치하여 조작 흐름 단순화

색상 대비를 통해 뜻이 보이는 상태 / 숨김 상태 구분

Material Design 가이드라인 준수

UI는 화려함보다 학습에 집중할 수 있는 구조를 우선으로 설계했습니다.
